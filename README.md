# Kala Nidhi by Rajeev Verma

Static MVP website for `kalanidhibyrajeev.com`.

The site is plain HTML, CSS, SVG, JSON, and minimal vanilla JavaScript. There is no framework, build step, package install, or deploy dependency.

## Site Routes

The website is now structured as a small static site:

```text
/
/about/
/art/
/books/
/workshops/
/daily-specials/
/visit/
```

The homepage is only the entry page. Art, books, workshops, daily specials, and visit details live on their own URLs.

## Preview Locally

From this folder:

```sh
python3 -m http.server 4173
```

Open:

```text
http://127.0.0.1:4173/
```

Use a local server instead of opening `index.html` directly, because the gallery, books, and workshops are loaded from JSON files.

## Deploy With Cloudflare Pages

1. Upload this project to a GitHub repository.
2. In Cloudflare Pages, create a new project from that repository.
3. Use:
   - Framework preset: `None`
   - Build command: leave empty
   - Output directory: `/`
4. Add the custom domain: `kalanidhibyrajeev.com`
5. Follow Cloudflare's DNS instructions for the domain.

## Deploy With GitHub Pages

1. Push this project to GitHub.
2. Open repository settings.
3. Go to Pages.
4. Enable Pages from the `main` branch and `/root`.
5. Keep the `CNAME` file in the root. It contains:

```text
kalanidhibyrajeev.com
```

6. Configure the DNS records required by GitHub Pages for the domain.

## Artwork Images

The WhatsApp artwork export has been imported into:

```text
assets/artworks/
```

The current gallery uses stable filenames from `artwork-001.jpg` through `artwork-089.jpg`.

To add, remove, rename, price, or recategorize artworks, edit:

```text
data/artworks.json
```

Each artwork record can include:

```json
{
  "title": "Inner Radiance",
  "category": "Mandalas",
  "medium": "Ink and acrylic on paper",
  "status": "Available",
  "image": "assets/artworks/mandala-001.jpg",
  "alt": "Inner Radiance mandala artwork by Rajeev Verma",
  "size": "18 x 24 inches",
  "price": "On request"
}
```

Current gallery categories are:

- Mandalas
- Oil Paintings
- Portraits
- Spiritual Art
- Decorative Art
- Custom Orders

If an image file is missing, the website shows a styled placeholder instead of a broken image.

## Add Book Covers

The current `data/books.json` contains 14 Amazon-linked Rajeev Verma listings found from the Amazon author search. Covers are still placeholders.

1. Add book cover images into:

```text
assets/books/
```

2. Update `data/books.json` with the real filename and Amazon link.

Example:

```json
{
  "title": "The Man Who Chose To Smile",
  "subtitle": "Short Stories for Youth",
  "author": "Rajeev Verma",
  "description": "Short stories for young readers by Rajeev Verma.",
  "link": "https://www.amazon.in/Chose-Smile-Short-Stories-Rajeev/dp/B0GTMJ237N/",
  "cover": "assets/books/the-man-who-chose-to-smile.jpg",
  "alt": "Cover of The Man Who Chose To Smile by Rajeev Verma"
}
```

## Update Workshops

Edit:

```text
data/workshops.json
```

Each workshop record controls the title, duration, audience, learning points, and WhatsApp enquiry message generated by the website.
